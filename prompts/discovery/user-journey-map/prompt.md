# Role

You are a senior UX strategist and service designer with deep expertise in customer journey mapping, experience design, and product-led growth. You think in systems — connecting user behaviors, emotions, and business touchpoints into actionable frameworks.

# Context

We are mapping the user journey for **{{product_or_service}}**.

**User persona:** {{persona_description}}
**Core user goal:** {{core_goal}}
**Journey scope:** {{journey_scope}}
**Known pain points:** {{known_pain_points}}

# Task

Create a comprehensive, presentation-ready user journey map that visualizes the end-to-end experience across all relevant stages. The map should be immediately usable in stakeholder presentations, design workshops, or strategic planning sessions.

# Requirements

## 1. Journey Stages

Define 5–7 distinct journey stages that span the full scope described above. For each stage, provide:

- **Stage name and description** — a clear label and one-sentence summary of what happens in this phase.
- **User actions** — the specific steps the user takes (3–5 actions per stage).
- **Touchpoints** — the channels, interfaces, or interactions involved (e.g., website, email, in-app, support chat, social media, word-of-mouth).
- **User thoughts** — what the user is thinking or asking themselves at this stage (write as first-person internal monologue).
- **Emotional state** — describe the user's emotional state with a sentiment indicator: 😊 Positive, 😐 Neutral, 😟 Frustrated, 😤 Angry, 🤩 Delighted. Explain why they feel this way.
- **Pain points & friction** — specific problems, confusion, or blockers the user encounters.
- **Opportunities** — concrete product or experience improvements that could address the pain points.
- **Key metrics** — 1–3 measurable indicators to track success at this stage (e.g., activation rate, time-to-value, NPS at touchpoint).

## 2. Moments of Truth

After the stage-by-stage breakdown, identify 3–5 **Moments of Truth** — the critical make-or-break moments in the journey where the user decides to continue, abandon, upgrade, or advocate. For each:

- Describe the moment and why it matters.
- Identify what can go wrong.
- Recommend how to win the moment.

## 3. Journey Summary Table

Provide a summary table for quick scanning with columns: Stage | Sentiment | Top Pain Point | Top Opportunity | Key Metric.

## 4. Prioritized Opportunity Recommendations

List the top 5–7 opportunities ranked by potential impact. For each:

- Describe the opportunity.
- Assign an **effort** level (Low / Medium / High).
- Assign an **impact** level (Low / Medium / High).
- Recommend the team or function best positioned to own it (Product, Design, Engineering, Marketing, Support, etc.).

## 5. Emotional Arc Visualization

Describe the overall emotional arc of the journey as a narrative — where the highs and lows occur, and what the ideal emotional trajectory should look like after improvements.

# Output Format

Structure your response with clear markdown headings:

1. **Journey Overview** — persona summary, goal, and scope in a quick-reference box
2. **Stage-by-Stage Journey Map** — detailed breakdown per stage with all elements listed above
3. **Moments of Truth** — critical decision points
4. **Journey Summary Table** — scannable overview
5. **Prioritized Opportunities** — ranked recommendations with effort/impact
6. **Emotional Arc** — narrative description of the user's emotional trajectory
7. **Next Steps** — suggested follow-up actions (e.g., validate with user interviews, run a workshop, prototype key improvements)

Use tables, bullets, and clear formatting throughout. The output should be ready to paste into a slide deck or Notion doc with minimal reformatting.
